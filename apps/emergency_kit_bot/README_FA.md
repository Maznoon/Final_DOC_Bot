# ربات تلگرام فروش کیت آمادگی بحران شهری (مدل اقتصادی)

این ربات یک سیستم کامل تجارت الکترونیک برای فروش کیت‌های آمادگی بحران ۷۲ ساعته است که با استفاده از Python و کتابخانه aiogram طراحی شده است.

## ویژگی‌ها

*   **رابط کاربری کاملاً فارسی:** تمامی متون، دکمه‌ها و اعداد به صورت فارسی و راست‌چین نمایش داده می‌شوند.
*   **جادوگر ثبت سفارش (Order Wizard):** فرآیند گام‌به‌گام برای دریافت اطلاعات مشتری، انتخاب استان (با قابلیت پیمایش) و تایید نهایی.
*   **سیستم پرداخت منعطف:** پشتیبانی از پرداخت کارت‌به‌کارت (با قابلیت آپلود رسید) و درگاه پرداخت آنلاین (بصورت شبیه‌سازی شده).
*   **پنل مدیریت قدرتمند:**
    *   مشاهده و مدیریت سفارش‌ها.
    *   تایید یا رد رسیدهای پرداخت کارت‌به‌کارت.
    *   تغییر قیمت و موجودی محصولات به صورت آنی.
    *   پاسخگویی مستقیم به پیام‌های پشتیبانی کاربران از طریق ربات.
*   **پیگیری سفارش:** امکان پیگیری وضعیت سفارش با استفاده از کد پیگیری ۶ رقمی.
*   **امنیت:** استفاده از متغیرهای محیطی برای تنظیمات حساس و محدودسازی دسترسی پنل ادمین.

## تکنولوژی‌های مورد استفاده

*   **زبان:** Python 3.11+
*   **کتابخانه ربات:** aiogram 3.x
*   **پایگاه داده:** PostgreSQL
*   **ORM:** SQLAlchemy 2.0 (Async)
*   **استقرار:** Docker & docker-compose

## راهنمای نصب و راه‌اندازی

### ۱. پیش‌نیازها
*   نصب بودن Docker و docker-compose
*   دریافت توکن ربات از @BotFather

### ۲. تنظیمات محیطی
فایل `.env.example` را به `.env` تغییر نام داده و مقادیر مربوطه را وارد کنید:
```env
BOT_TOKEN=توکن_ربات_شما
ADMIN_USER_IDS=12345678,87654321
DATABASE_URL=postgresql+asyncpg://user:password@db:5432/emergency_kit_db
SUPPORT_CONTACT=@آیدی_پشتیبانی
CARD_OWNER_NAME=نام_صاحب_کارت
CARD_NUMBER=شماره_کارت_۱۶_رقمی
```

### ۳. اجرا با Docker (پیشنهادی)
برای اجرای سریع و بدون دردسر، از دستور زیر استفاده کنید:
```bash
docker-compose up -d --build
```
این دستور پایگاه داده PostgreSQL و خودِ ربات را به صورت خودکار راه‌اندازی می‌کند.

### ۴. اجرای محلی (Local)
در صورتی که می‌خواهید بدون داکر اجرا کنید:
1. ایجاد محیط مجازی و نصب وابستگی‌ها:
   ```bash
   python -m venv venv
   source venv/bin/activate
   pip install -r requirements.txt
   ```
2. مقدار `DATABASE_URL` را در `.env` به دیتابیس محلی خود تغییر دهید.
3. اجرای ربات:
   ```bash
   python -m src.main
   ```

### ۵. مقداردهی اولیه محصولات (Seed)
برای اضافه کردن محصولات اولیه به دیتابیس:
```bash
docker-compose exec bot python -m src.db.seed
```

## ساختار پروژه
```
src/
├── config/        # تنظیمات برنامه
├── db/            # مدل‌های دیتابیس و مدیریت نشست‌ها
├── handlers/      # هندلرهای پیام‌ها و دستورات (منو، سفارش، ادمین و ...)
├── services/      # سرویس‌های کمکی (پرداخت، کاربر)
├── utils/         # ابزارهای کمکی (تبدیل اعداد، کیبوردها، استیت‌ها)
└── main.py        # نقطه ورود برنامه
```
